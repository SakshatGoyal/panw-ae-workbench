# Link

[Source code](packages/link/src/Link/Link.tsx)

## Overview

Inline text link with two color palettes (`blue`, `black`), two sizes (`14px`,
`18px`), optional leading/trailing arrow icons, and a disabled state. Renders
as `<a>` when `href` is provided, otherwise as a `<span role="link">`.

## Accessibility

- Renders as `<a>` when `href` is provided — native focus, activation, and
  screen reader announcement work out of the box.
- Without `href`, renders as `<span role="link">` for click-only handlers.
- Disabled links use `aria-disabled="true"` and `tabIndex={-1}` so they remain
  in the DOM but are not focusable or interactive.
- Icons (`leftIcon`, `rightIcon`) are wrapped in a span with `aria-hidden="true"`.

## API Change: `renderLeftIcon` / `renderRightIcon`

PANW's source rendered fixed inline arrow SVGs. The port keeps `leftIcon` /
`rightIcon` as the boolean toggle (matches PANW behavior) but exposes
`renderLeftIcon` / `renderRightIcon` as `ElementType` overrides, defaulting to
lucide-react `ArrowLeft` / `ArrowRight`.

```jsx
import { ChevronRight } from 'lucide-react';

<Link href="#" rightIcon renderRightIcon={ChevronRight}>Continue</Link>
```
