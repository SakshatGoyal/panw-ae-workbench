## Overview

Buttons are clickable elements used to trigger actions. They communicate calls
to action and allow users to interact with pages in a variety of ways. Button
labels express what action will occur when the user interacts with it.

## Kinds

PANW Button has six kinds. `primary` and `danger` map directly to Carbon's
equivalent kinds. The remaining four (`secondary-accent`, `secondary-gray`,
`ghost-accent`, `ghost-gray`) are PANW-specific extensions with no direct
Carbon equivalent — see [PANW Extensions](#panw-extensions) below.

## With Icons

Pass a lucide-react (or any React) component to `renderIcon`. The icon renders
in the `left` or `right` position relative to the label, controlled by
`iconPosition`.

<Unstyled>
  <div style={{ display: 'flex', gap: '8px', padding: '16px' }}>
    <Button
      renderIcon={(props) => <Plus size={16} {...props} />}
      iconPosition="left"
      iconDescription="Add"
    >
      Add Item
    </Button>
  </div>
</Unstyled>

```jsx
import { Plus } from 'lucide-react';

<Button
  renderIcon={(props) => <Plus size={16} {...props} />}
  iconPosition="left"
  iconDescription="Add"
>
  Add Item
</Button>
```

## Accessibility

- The `<button>` element is used by default, which provides native keyboard
  focus, `Enter`/`Space` activation, and screen reader role announcement.
- When `href` is provided, the component renders as `<a>` — add `role="button"`
  only if you need button semantics on a link.
- When `renderIcon` is used without children, provide `iconDescription` so
  screen readers can announce the icon purpose.
- Disabled buttons use the native HTML `disabled` attribute, which removes
  them from the tab order. This is intentional — disabled actions should not
  receive focus. {/* TODO: confirm PANW's accessibility guidance on disabled focus */}

## API Change: `renderIcon`

**Deliberate change from PANW `ButtonStandard`:**

PANW's `ButtonStandard` accepted `leftIconSlot` and `rightIconSlot` as
`ReactNode` — you passed already-rendered elements:

```jsx
// PANW (old)
<ButtonStandard leftIcon leftIconSlot={<MyIcon />} label="Add" />
```

The ported `Button` uses Carbon's `renderIcon: ElementType` convention — you
pass the component class/function, not a rendered element:

```jsx
// Ported (Carbon convention)
<Button renderIcon={(props) => <Plus size={16} {...props} />} iconPosition="left">
  Add
</Button>
```

This change enables the component to control `aria-hidden`, size, and class
injection on the icon — which is not possible when a ReactNode is passed in.
