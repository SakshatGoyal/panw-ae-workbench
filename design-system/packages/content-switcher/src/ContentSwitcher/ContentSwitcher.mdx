# ContentSwitcher

[Source code](packages/content-switcher/src/ContentSwitcher/ContentSwitcher.tsx)

## Overview

A segmented control / tablist used to switch between two or more peer views.
Controlled by `selectedIndex` and `onChange`. Built on `role="tablist"` with
`role="tab"` segments and roving tabindex.

  items={[{ label: 'List' }, { label: 'Grid' }, { label: 'Calendar' }]}
  selectedIndex={idx}

## Accessibility

- Container has `role="tablist"`, segments have `role="tab"` and
  `aria-selected` reflecting state.
- Roving `tabIndex`: only the selected segment has `tabIndex={0}`.
- Each segment is a real `<button type="button">` so Enter/Space activate it.
- Disabled propagates to the native `disabled` attribute on every segment.

## API Change: `icon` (ReactNode) → `renderIcon` (ElementType)

PANW's `ContentSwitcherItem.icon` accepted a `ReactNode`. The port renames it
to `renderIcon: ElementType` to match Carbon's icon convention. Pass the
component, not a rendered element:

```jsx
import { List } from '@ds/icons';

<ContentSwitcher
  items={[{ label: 'List', renderIcon: List }]}
  selectedIndex={0}
/>
```
