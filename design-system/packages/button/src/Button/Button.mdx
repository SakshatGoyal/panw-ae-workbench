## Overview

Buttons trigger actions. The label states what action will occur.

Stage's call: five canonical kinds, one corner (radius.tight = 4px). Brand
intent is encoded in the kind name — there is no `-accent` or `-gray` suffix
to manage. Decide which kind expresses the role, the visual treatment follows.

## Kinds

Five kinds map to ascending tiers of emphasis. Use one primary per surface;
reach for tertiary when you want a brand-tinted action that is not the lead;
ghost is the ambient neutral; danger is reserved for destructive operations.

| Kind        | Treatment                                  | Role                              |
|-------------|--------------------------------------------|-----------------------------------|
| `primary`   | Brand fill, inverse text                   | Lead call to action               |
| `secondary` | Surface bg, neutral text, neutral border   | Supporting action                 |
| `tertiary`  | Surface bg, brand text, neutral border     | Soft brand action                 |
| `ghost`     | Transparent, neutral text                  | Ambient / inline action           |
| `danger`    | Red fill, inverse text                     | Destructive operation             |

## Disabled

Disabled is content-led for `ghost` and `tertiary` — the surface stays put,
only text dims. `primary`, `secondary`, and `danger` dim the surface itself.

## Accessibility

- The `<button>` element is used by default; native keyboard focus and
  `Enter`/`Space` activation are preserved.
- When `href` is provided, the component renders as `<a>` — add `role="button"`
  only if button semantics on a link are needed.
- When `renderIcon` is used without children, provide `iconDescription` so
  screen readers announce the icon purpose.
- Disabled buttons use the native HTML `disabled` attribute, which removes
  them from the tab order.
