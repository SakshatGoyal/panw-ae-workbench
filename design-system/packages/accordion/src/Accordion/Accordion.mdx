# Accordion

[Source code](packages/accordion/src/Accordion/Accordion.tsx)

## Overview

A single, expandable disclosure with optional leading icon, supplemental
description, and an inline tag. Three sizes (`small`, `default`, `large`),
two themes (`gray00`, `gray10`), and a right-side orientation.

## Variants

### Sizes

```jsx
<Accordion size="small" title="Small" />
<Accordion size="default" title="Default" />
<Accordion size="large" title="Large" />
```

### Theme

```jsx
<Accordion theme="gray10" title="Gray 10 background" />
```

### Orientation

```jsx
<Accordion orientation="right" title="Chevron right" />
```

## Composition

The optional inline tag is rendered using the `@ds/tags` package — it is
imported, not redefined.

```jsx
<Accordion showTag tagLabel="Beta" tagColor="green" title="With tag" />
```

## Accessibility

- Header has `role="button"`, `aria-expanded`, `aria-disabled`.
- `Enter` and `Space` toggle when focused.
- `tabIndex` is `-1` when disabled.
- Icons are `aria-hidden="true"`.

## API Change: `customIcon` → `renderIcon`

PANW originally accepted `customIcon: ReactNode`. This port follows the
Carbon convention: `renderIcon: ElementType` so the component controls
sizing and `aria-hidden`.

```jsx
import { Star } from '@ds/icons';
<Accordion showIcon renderIcon={Star} title="With icon" />
```
