## Overview

IconButton is an icon-only button — no visible label text. It is used when
space is constrained and the icon is universally understood, or when the action
is supplementary to nearby context. Every `IconButton` requires an `aria-label`
for screen reader accessibility.

## Sizes

Three sizes: `sm` (32px), `md` (40px), `lg` (48px). Heights are padding-based;
the icon pixel size is controlled separately via `iconSize`.

## Accessibility

- `aria-label` is **required** — there is no visible text to announce.
  Provide a concise, action-oriented label (e.g. `"Close dialog"`, `"Add item"`).
- The icon is wrapped in `aria-hidden="true"` so screen readers skip it.
- The `disabled` attribute removes the button from the tab order. {/* TODO: confirm PANW a11y guidance */}
- `isSelected` on ghost buttons sets `aria-pressed` to expose the toggle
  state to assistive technology.

## API Change: `renderIcon`

**Deliberate change from PANW `ButtonIcon`:**

PANW's `ButtonIcon` accepted `icon: ReactNode` — a pre-rendered element:

```jsx
// PANW (old)
<ButtonIcon icon={<MyIcon />} aria-label="Add" />
```

The ported `IconButton` uses Carbon's `renderIcon: ElementType` convention:

```jsx
// Ported (Carbon convention)
import { Plus } from '@ds/icons';

<IconButton renderIcon={(props) => <Plus size={16} {...props} />} aria-label="Add" />
```

This aligns with the `Button` component's `renderIcon` API and enables
consistent icon handling across the system.
